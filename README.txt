STEP 1: DISTRIBUTED DEPLOYMENT MODEL (Edge / Cloud / Dashboard)
------------------------------------------------------------------------------------------------------------
create and install the following packages to their respective machines/VMs.

+-------------------+---------------------------------------------+----------------------------------------+
| PACKAGE NAME      | WHAT TO INCLUDE                             | PURPOSE / NOTES                        |
+-------------------+---------------------------------------------+----------------------------------------+
| Cloud-package     | cloud/  common/  requirements.txt           | offload/install on the cloud machine/VM|
|                   |                                             | Start `cloud/server.py`.               |
|                   |                                             | `common/` must be present.             |
+-------------------+---------------------------------------------+----------------------------------------+
| Edge-package      | edge/  common/  data/  dashboard/  output/  | Put on the edge machine/VM.            |
|                   |  requirements.txt                           | Orchestrator `edge/orchestrator.py`    |
|                   |                                             | writes `output/metrics_history.json`.  |
+-------------------+---------------------------------------------+----------------------------------------+
| Dashboard         | dashboard/                                  | Can run on edge machine/VM or any      |
|                   |                                             | machine with access to                 |
|                   |                                             | `output/metrics_history.json`          |
+-------------------+---------------------------------------------+----------------------------------------+

NOTE: `common/` and `requirements.txt` must be included in both packages.
      `output/` if not copied the orchestrator will create it, belongs to the edge package.
      `experiments/` cantinas results and analysis of the experiments conducted.

Example cloud: tar -czf cloud_bundle.tar.gz cloud common requirements.txt
Example edge: tar -czf edge_bundle.tar.gz edge common data dashboard output requirements.txt

------------------------------------------------------------------------------------------------------------
STEP 2: RUNNING THE SYSTEM
------------------------------------------------------------------------------------------------------------

2.1 CREATE & ACTIVATE VENV: ON EACH MACHINE
------------------------------------------------------------------------------------------------------------
Make sure that the working directory at the same level as the package folder.

python3 -m venv venv
source venv/bin/activate



2.2 EXTRACT PACKAGES & INSTALL DEPENDENCIES: BOTH MACHINES
------------------------------------------------------------------------------------------------------------
# then extract packages. example: tar -xvzf package_name.tar.gz

pip3 install --upgrade pip

python -m pip install -r requirements.txt



2.3 START THE CLOUD SERVER: CLOUD MACHINE
------------------------------------------------------------------------------------------------------------
# then from the package root: one level above cloud/

python -m cloud.server --port 5000

# by default cloud server binds to all interfaces: host='0.0.0.0', so the edge can reach it.



2.4 START THE DASHBOARD: EDGE MACHINE
------------------------------------------------------------------------------------------------------------
# after that and from the package directory: one level above dashboard/

streamlit run dashboard/dashboard.py

# Dashboard waits for: output/metrics_history.json



2.5 RUN THE EDGE ORCHESTRATOR: EDGE MACHINE
------------------------------------------------------------------------------------------------------------
# from the package root: one level above edge/

python -m edge.orchestrator --video_path "data/experiment_sample.mp4" --server_url "http://127.0.0.1:5000"


# Add --store_video to store the annotated video to the output folder instead of realtime display. In VMs without GUI void it i.e. headless VMs.
# if required change the --server_url to match the cloud host and port.



------------------------------------------------------------------------------------------------------------
STEP 3. EXPERIMENT RUNS
------------------------------------------------------------------------------------------------------------
3.1 CLOUD-ONLY: force offload

python -m edge.orchestrator --video_path "data/experiment_sample.mp4" --skip_interval 0 --heuristic_threshold 5.0 --edge_conf_threshold 1.1 --m_output_dir "experiments/cloud_only" --server_url "http://10.0.0.3:5000"



3.2 EDGE-ONLY: force EDGE

python -m edge.orchestrator --video_path "data/experiment_sample.mp4" --skip_interval 0 --heuristic_threshold 5.0 --edge_conf_threshold -1 --m_output_dir "experiments/edge_only" --server_url "http://10.0.0.3:5000"


IMPORTANT For EDGE-ONLY: to avoid offloading frames to the cloud when edge model reports no detections e.g. empty detection, change the decision gateway logic
in edge/edge_model.py (i.e. the decision method edg_model_decision()):

Replace:
    if not confidences_list:
        True (i.e. send_to_cloud)
with:
    if not confidences_list:
        False (i.e. keep_local)

Then change it back after experiment



3.3 FILTER ON / OFF

--heuristic_threshold 5.0  # ON (default)
--heuristic_threshold 0     # OFF


--------------------------------------------------------------------------------------------------------------
STEP 4. WHERE RESULTS APPEAR
--------------------------------------------------------------------------------------------------------------

- output/metrics_history.json    : per frame performance metrics including edge network, cloud and content info
- output/annotated_output.mp4   : annotated video output, when saving is enabled, default behaviour
- experiments/experiments.ipynb : analysis plots and observations

+----------------------------+--------+----------------------+---------------------------------------------------------------------+
| FLAG                       | TYPE   | DEFAULT              | MEANING                                                             |

| --server_url               | str    | http://10.0.0.3:5000 | URL for the cloud server the POST endpoint.                         |

| --resize_width             | int    | 640                  | Resizing width (px)                                                 |
| --quality                  | int    | 80                   | JPEG quality in the rage [0â€“100].                                   |
| --heuristic_threshold      | float  | 5.0                  | motion  threshold. Greyscale mean abs diff threshold [0â€“255].       |
| --edge_conf_threshold      | float  | 0.70                 | Edge model decision /acceptance threshold [0â€“1].                    |
| --m_output_dir             | str    | output               | Output directory for metrics and video.                             |
| --debug_mode               | flag   | False                | Enable the mode of debugging , that processes limited frames        |
